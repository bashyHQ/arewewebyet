External Services

The modern web development stack doesn't only need a web-server but is often built on a range of external services to provide specific features, from worker queues to search and pubsub. Rust support for these is certainly not the greatest, but there are popular crates for the most popular services.

Cloud SDK

  • rusoto_core

    AWS SDK for Rust - Core

  • azure_sdk_for_rust

    Rust wrappers around Microsoft Azure REST APIs

Worker Queue

  • amqp

    AMQP/RabbitMQ protocol client

  • beanstalkd

    Easy-to-use beanstalkd client for Rust (IronMQ compatible)

  • celery

    Rust implementation of Celery

  • kafka

    Rust client for Apache Kafka

  • lapin

    AMQP client library

  • stomp

    A full STOMP 1.2 client implementation. Allows programs to interact with message queueing services like ActiveMQ and RabbitMQ.

Query engines

  • datafusion

    DataFusion is an in-memory query engine that uses Apache Arrow as the memory model

Search Services

  • elasticsearch

    Official Elasticsearch Rust client

  • meilisearch-sdk

    Rust wrapper for the MeiliSearch API. MeiliSearch is a powerful, fast, open-source, easy to use and deploy search engine.

  • tantivy

    Search engine library

Pubsub

  • pulsar

    Rust API bindings for Apache Pulsar

  • redis

    Redis driver for Rust.

Connectors

  • efflux

    Easy MapReduce and Hadoop Streaming interfaces in Rust
